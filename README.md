# 지역 문화 격차 해소를 위한 'SAC on Screen' 최적지 도출 및 맞춤형 콘텐츠 매칭 시스템

## 1. 문제 정의 (Problem Definition)

대한민국의 대형 공연 및 전시 중 70% 이상이 수도권에 집중되어 있어, 지방 거주민들이 느끼는 문화적 소외감과 지역 간 문화 격차는 갈수록 심화되고 있습니다. 이러한 불균형을 해소하기 위해 예술의전당(Seoul Arts Center)은 고품질의 문화예술 공연 실황을 영상화하여 전국 각지의 문화시설에서 상영하는 **'SAC on Screen'** 사업을 운영하고 있습니다. 그러나 기존 사업은 주로 개별 시설의 단편적인 신청에 의존하거나, 지역의 인구 통계학적 특성을 세밀하게 반영하지 못한 채 일률적으로 장르를 배분하여 운영 효율성이 낮다는 한계가 있습니다.

본 프로젝트는 예술의전당의 실제 공연 예매 데이터와 전국의 지역별 인구·문화 인프라 데이터를 융합하여, **"어느 지역에 어떤 공연을 상영하는 것이 가장 효과적인가?"**라는 질문에 답하는 데이터 기반의 의사결정 지원 시스템을 제안합니다. 이를 통해 한정된 예산과 자원 내에서 지역 주민들의 실제 문화 수요(연령대, 선호도)를 정교하게 반영한 최적의 상영지와 맞춤형 콘텐츠 매칭 전략을 도출하고자 합니다.

### 1.2 핵심 분석 질문 (Analysis Questions)

본 프로젝트는 PySpark 및 HiveQL을 통해 대규모 데이터를 조인 및 집계하여 다음 세 가지 핵심 질문에 대한 정량적 해답을 도출합니다.

1. **연령대별 선호 콘텐츠와 킬러 장르는 어떻게 분포하는가?**
   - 예술의전당 예매 데이터를 기반으로 연령대(10대~60대 이상)에 따른 장르별 예매 비중과 키워드 연관성을 분석하여 콘텐츠-연령 선호도 맵핑 테이블을 구축합니다.
2. **지역별 잠재적 문화 수요 지수(Demand Index)는 어떻게 산출되는가?**
   - 시/군/구 단위의 연령 분포 데이터와 연령대별 콘텐츠 선호도 가중치를 결합하여, 각 지역이 가진 특정 장르에 대한 잠재 수요를 정량적으로 계산합니다.
3. **접근성 및 인프라를 고려한 'SAC on Screen'의 최적 우선순위 입지는 어디인가?**
   - 문화 시설 인프라가 취약한 '문화 사각지대'를 정의하고, 잠재 수요 지수가 높으면서도 상영 가능 시설을 보유한 최적의 지역 및 시설 리스트를 도출합니다.

## 2. 기술 스택 (Technology Stack)

강의에서 다룬 Hadoop 생태계와 빅데이터 처리 프레임워크를 유기적으로 연결하여 안정적인 데이터 파이프라인을 구축합니다.

- **Data Ingestion (수집):** Python (Requests, BeautifulSoup)을 사용하여 문화데이터광장 및 공공데이터포털의 API와 파일 데이터를 수집하고 정형화합니다.
- **Data Storage (저장):** 수집된 대용량 Raw 데이터를 **Hadoop Distributed File System (HDFS)**에 분산 적재하여 안정성을 확보합니다.
- **Data Processing (전처리 및 분석):** **Apache Spark (PySpark & Spark SQL)**을 활용하여 대규모 데이터 조인, 연령별 선호 가중치 계산, 지역별 수요 지수 산출 및 랭킹 알고리즘을 분산 처리합니다.
- **Data Warehouse (데이터 웨어하우스):** **Apache Hive**를 활용하여 Spark의 처리 결과를 Parquet 포맷의 External Table로 관리하고, HiveQL을 통해 다차원 분석 쿼리를 수행합니다.
- **Workflow Automation (자동화):** **Apache Airflow** DAG 또는 Shell Script 파이프라인을 구축하여 수집부터 적재, 처리까지의 과정을 단일 흐름으로 제어합니다.
- **Visualization (시각화):** 분석 완료된 데이터를 **Streamlit** 대시보드 및 **Matplotlib / Seaborn** 라이브러리를 통해 지도 및 차트 형태로 시각화합니다.

## 3. 구현 계획 (Implementation Plan)

데이터 수집부터 가공, 분석, 시각화에 이르는 엔드투엔드(End-to-End) 파이프라인을 단계별로 나누어 구현합니다.

```text
[공공 API / 공공데이터포털] 
       │ (Python Ingestion Script)
       ▼
[Local Raw Data (CSV/JSON)]
       │ (HDFS Upload Script)
       ▼
[HDFS (/user/ubuntu/sac_on_screen/raw/)]
       │ (PySpark 분산 처리 & Feature Engineering)
       ▼
[HDFS Processed Data (Parquet)]
       │ (Hive External Table 등록)
       ▼
[HiveQL / Spark SQL 다차원 분석]
       │ (Pandas / Streamlit 연동)
       ▼
[시각화 대시보드 & 최적 입지 추천]
```

### 3.1 1단계: 데이터 수집 및 HDFS 적재 (Ingestion)
- 예술의전당 예매/공연 API 및 지역 인구 CSV 데이터를 수집하는 Python 스크립트를 작성합니다.
- 데이터 정규화 및 결측치 처리를 수행한 후, 로컬 디렉토리에 저장합니다.
- 작성된 쉘 스크립트(`upload_to_hdfs.sh`)를 실행하여 HDFS의 `/user/ubuntu/sac_on_screen/raw/` 경로에 도메인별로 구분하여 업로드합니다.

### 3.2 2단계: PySpark 기반 분산 전처리 및 피처 엔지니어링 (Processing)
- HDFS의 Raw 데이터를 PySpark DataFrame으로 로드합니다.
- 예술의전당 예매 건수 데이터를 기반으로 **연령대별 선호도 가중치**를 계산합니다.
- 지역별 인구 데이터와 문화 시설 정보를 행정구역(시군구) 기준으로 조인하여 **지역별 수요 지수(Demand Index)**를 산출합니다.
- 최종 가공된 데이터를 쿼리 성능 최적화를 위해 Parquet 포맷으로 HDFS의 `/user/ubuntu/sac_on_screen/processed/`에 저장합니다.

### 3.3 3단계: Hive 데이터 웨어하우징 및 심층 분석 (Analytics)
- HDFS에 저장된 Parquet 데이터를 기반으로 Hive External Table을 생성합니다.
- HiveQL을 통해 장르별 인기 순위, 지역별 잠재 수요 순위, 문화 소외 지역 내 상영관 인프라 현황 등을 다차원 분석합니다.
- 분석 쿼리 결과를 요약 테이블로 구성하여 최종 대시보드용 데이터로 활용합니다.

### 3.4 4단계: 대시보드 시각화 및 자동화 (Visualization & Automation)
- Streamlit을 활용하여 전국의 '문화 활력 및 수요 지도'를 인터랙티브하게 시각화합니다.
- 사용자가 특정 지역을 선택하면 해당 지역의 인구 구조에 특화된 예술의전당 추천 공연 목록(맞춤형 큐레이션)을 보여주는 화면을 구현합니다.
- 이 모든 과정이 매끄럽게 재현될 수 있도록 전체 파이프라인 구동 쉘 스크립트를 제공합니다.

## 4. 확장 가능성 및 신중한 설계 고려사항

본 프로젝트는 추후 실제 사업에 적용하거나 데이터를 추가 적재할 때 발생할 수 있는 변화를 대비하여 다음과 같이 유연하게 설계되었습니다.

- **데이터 추가 및 스키마 변경 대비:** 공연 정보 및 인구 통계 데이터의 필드가 추가되거나 변경되더라도 파이프라인이 중단되지 않도록 Spark 스키마 옵션에 `mergeSchema`를 적용하고, Hive External Table 구조를 채택하여 데이터와 스키마를 독립적으로 관리합니다.
- **분석 알고리즘의 고도화 여지:** 현재의 인구 비례 수요 지수 산출 방식에서 나아가, 향후 머신러닝 기반의 클러스터링(K-Means)이나 추천 알고리즘(ALS)을 쉽게 플러그인 형태로 추가할 수 있도록 `src/modeling/` 디렉토리 구조를 분리하여 설계했습니다.
- **유연한 데이터 수집 범위:** 예술의전당 데이터뿐만 아니라 다른 국립 예술단체의 데이터가 추가되더라도 수집 스크립트만 모듈식으로 추가하면 전체 파이프라인에 즉시 융합될 수 있도록 설계 구조를 표준화했습니다.
